## Brand Story

- 고양이를 키우는 팀원과 고양이를 좋아하는 팀원들이 모여 냥빌리지가 탄생했습니다.
- 고양이를 키우고 싶은 사람들에게는 정보를 제공하고 고양이를 키우고 있는 사람들에겐 고양이 사진을 공유할 수 있는 공간을 제공하려는 목적으로 만들었습니다.
- 저희팀은 고양이를 좋아하는 팀원들을 위해 고양이 사진을 공유하면서 바쁜 프로젝트 일정속에 잠시나마 힐링하는 시간을 가져왔습니다. 이 경험을 살려 저희는 사용자에게 힘든 현실은 잠시 잊고 편안한 느낌을 드리는 서비스를 만들고자 기획하였습니다.
- 바쁘고 스트레스에 절은 현대인에게 귀엽고 사랑스러운 고양이 사진을 보면서 **🍀힐링**할 수 있는 공간을 제공하려는 목적으로 만들었습니다.

### Target

- 고양이를 키우는 사람들
- 고양이를 키우지 않지만 좋아하는 사람들
- 고양이를 좋아하는 개발자들

✅ *사진을 게시하는 SNS* 특성 상 사용자가 **모바일**로 편하게 볼 수 있도록 모바일을 기준으로 화면을 설계하였습니다.

✅ 모바일에 맞춰 디자인을 심플하면서도 깔끔하게 설계하였습니다.

✅ 사용자에게 편안한 느낌을 주는 디자인으로 설계하였습니다.

# 서비스 소개

이메일과 비밀번호, 닉네임만으로 간단하게 냥빌리지의 주민이 되보세요! 👥

🐈 고양이를 등록하고 자신만의 고양이 프로필을 만들어보세요.

고양이 프로필에 사랑스런 고양이 사진과 함께 태그와 설명을 추가해서 자유롭게 꾸밀 수 있습니다.

👩🏻‍🦰 집사 프로필에 프로필 사진을 등록해서 나만의 프로필을 꾸며보세요!

사진이 없어도 걱정마세요! 사진을 올리지 않으면 자동으로 기본 프로필 사진이 적용됩니다.

냥빌리지의 주민은 총 2가지의 서로 다른 성격의 서비스를 이용하실 수 있습니다.

✅ 냥이생활은 고양이 사진을 게시하며 자랑할 수 있는 SNS 공간입니다.

✅ 집사생활은 유저가 자유롭게 질문과 답변을 올릴 수 있는 커뮤니티 입니다.

✅ 집사생활에서는 고양이 인기 품종의 성격, 털길이, 털빠짐 정도, 외형 등의 정보와 함께 고양이가 비만인지 확인할 수 있는 BMI 정보가 준비되어 있습니다.

### 유저

- 회원가입하지 않은 사용자는 이용에 제한이 있습니다!
- 나만의 집사 프로필👤을 만들 수 있습니다!
- 고양이 프로필🐱을 최대 3개까지 등록할 수 있습니다!
- 고양이 프로필에 사진, 이름 뿐만 아니라 추가로 소개글과 태그도 작성할 수 있습니다!
- 마음에 드는 회원을 팔로우 할 수 있습니다.
- 내 프로필에서 나를 팔로우한 집사와 내가 팔로우한 집사를 볼 수 있습니다.
- 로그인하면 메인 피드에서 내가 팔로우한 고양이 프로필을 모아볼 수 있습니다.
- 궁금한 유저의 프로필에 들어가서 고양이와 게시글을 모아보세요!

### 냥이생활

- 내 고양이 프로필 중 하나를 선택해서 고양이 사진을 게시하고 자랑할 수 있는 게시판입니다.
- 냥이생활에 게시글 업로드는 고양이 프로필로 가능합니다.
- 하지만 댓글 작성은 고양이 프로필과 집사 프로필 중에 선택하여 작성할 수 있습니다.
- 게시글 당 사진📸을 1장만 게시할 수 있습니다. (추후 업로드 사진 개수 제한 업데이트 예정)
- 마음에 드는 사진에 좋아요♥️와 댓글💬을 추가해보세요!
- 댓글에도 좋아요♥️를 추가할 수 있습니다.
- 고양이와 집사 프로필을 이용해서 고양이와 집사가 대화하는 것처럼 댓글을 작성해보세요~🤣

 

### 집사생활

- 집사생활에서는 집사 프로필로만 활동할 수 있습니다.
- 태그를 선택하여 집사생활에 질문을 올리고 자유롭게 답변을 달아보세요!
- 집사생활에 게시글 작성 시 사진📸을 함께 첨부할 수 있습니다!
- 고양이의 품종별 정보를 제공 해드립니다!
- 고양이 BMI 정보, 예방접종 시기 정보 등을 제공합니다!
- 고양이에 대해서 궁금한 부분을 게시글로 작성할 수 있습니다!
- 게시글에 좋아요♥️ 와 댓글💬을 추가할 수 있습니다!
- 유용한 게시글이나 댓글에 좋아요♥️를 추가해보세요!

## branch 전략
| branch 이름 | 전략 |
|:--------:|:--------:|
| `main` 브랜치 | `dev` 브랜치에서 배포할 수 있을 정도로 구현된 것을 Merge 후 배포하는 브랜치 |
| `dev` 브랜치 | 기능 개발과 버그 수정 브랜치와 Merge가 자주 일어나는 브랜치, 개발이 가장 활발하게 일어나는 브랜치 |
| `feat/기능이름` or `fix/기능이름` | 이슈나 기능에 대해 개발하는 브랜치, 구현이 끝나면 `dev` 브랜치에 Merge |

브랜치 이름을 만들 때 되도록이면 대문자 쓰지 말기!! <br>
단어가 여러개 들어가면 하이폰(-)으로 연결하기 ex) feat/get-cat

<br><br>

## commit 전략
| commit 이름 | 전략 |
|:--------:|:--------:|
| `feat` | 새로운 기능 |
| `fix`  | 버그 수정 |
| `docs` | README, wiki 등 문서 수정 |
| `style` | 코드 변경 없이 스타일 변경 (CSS 파일 등) |
| `refactor` | 리팩토링 (클린 코드) |
| `chore` | 동작에 영향이 없는 코드 변경 없는 변경사항(주석 추가 등등) |

<br><br>

## Pull Request Merge 전략<br>
| source branch ➡️ definition branch | Merge 전략 |
|:--------:|:--------:|
| feat(or fix) ➡️ dev | Merge Commit |
| dev ➡️ main | Squash and Merge |
